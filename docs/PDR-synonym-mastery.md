# PDR: Synonym Mastery System
**작성일:** 2026-03-15  
**상태:** 검토 중

---

## 1. 문제 정의

현재 앱의 암기 흐름은 **단어 ↔ 뜻** 중심으로 설계되어 있어 동의어(synonym)가 실질적으로 훈련되지 않는다.

- 플래시카드: 단어 보고 뒤집기 → 뜻+동의어를 **수동으로 확인** (패시브)
- 타이핑 테스트: 뜻 보고 단어만 타이핑 → 동의어는 **완전히 제외**
- 쓰기 연습: 단어만 타이핑 → 동의어 **미포함**
- 3방향 테스트: 자가 평가 방식 → 동의어 **실제 출력 없음**

결과적으로 동의어는 "보는 것"에 그치며 능동적 리콜(active recall)이 없다.

---

## 2. 목표

> 단어, 뜻, 동의어 세 가지를 **동시에** 능동적으로 훈련할 수 있는 학습 흐름 구축

### 2-1. 쓰기 연습 목표
- 단어+뜻+동의어를 **보면서** 전부 타이핑
- 모터 메모리(근육 기억) 활용
- 동의어가 여러 개일 때 순서대로 입력 유도

### 2-2. 리콜 테스트 목표  
- 단서만 보고 동의어를 **직접 입력**하거나 자가 평가
- 동의어 여러 개 중 일부만 맞춰도 부분 정답 처리

---

## 3. 데이터 구조 분석

현재 CSV 동의어 필드 패턴:

| 패턴 | 예시 |
|------|------|
| 단순 나열 | `provide, give, grant` |
| 세미콜론 구분 | `probability, likelihood; opportunity` |
| 번호 구분 | `1. promise, contract 2. reserve, book` |
| 단일 | `full of` |

→ 동의어 파싱 전략 필요: 세미콜론/번호로 그룹 분리, 쉼표로 개별 항목 분리

---

## 4. 학습 모드 설계안

### 4-A. 쓰기 연습 (개선)
**현재:** 영단어만 타이핑  
**개선안:** 3단계 순차 입력

```
Step 1: [영단어 입력]    ← 단어 타이핑
Step 2: [뜻 입력]        ← 한국어 뜻 타이핑 (또는 확인 후 자가 평가)
Step 3: [동의어 입력]    ← 영어 동의어 타이핑 (쉼표 구분)
```

**문제점:**  
- 한국어 뜻 타이핑: IME 입력 방식 차이로 정확 비교 어려움  
- 동의어 순서: `provide, give, grant` vs `give, provide, grant` 모두 정답 처리 필요  
- 번호 구분 동의어(`1. promise... 2. reserve...`): 그룹별 최소 1개 입력 시 정답?

**결론:** 뜻은 **보여주고 확인** 방식, 동의어는 **타이핑** 방식이 현실적

---

### 4-B. 3방향 테스트 (개선)
**현재:** 자가 평가만  
**개선안:** 방향별 입력 필드 추가

| 방향 | 프롬프트 | 입력 방식 |
|------|---------|---------|
| 단어 → 뜻+동의어 | 단어 표시 | 자가 평가 (뜻은 한국어라 입력 비현실적) |
| 뜻 → 단어+동의어 | 뜻 표시 | 단어 타이핑 + 동의어 타이핑 |
| 동의어 → 단어+뜻 | 동의어 표시 | 단어 타이핑 + 자가 평가(뜻) |

---

### 4-C. 동의어 전용 타이핑 모드 (신규)
동의어 암기에 특화된 별도 모드

```
[단어 + 뜻 표시]
"provide, give, ___" 형태로 빈칸 채우기
또는
동의어 전체를 직접 타이핑
```

## 5. 동의어 정답 판정 로직

```
입력: "provide, give"
정답: "provide, give, grant"

판정 방식 옵션:
A. 전체 일치 (엄격) → UX 나쁨
B. 포함 비율 (2/3개 = 67%) → 일정 기준 이상 정답
C. 최소 1개 포함 → 너무 관대
D. 정답 표시 후 자가 평가 → 가장 현실적
```

→ **추천: D (자가 평가)** — 동의어는 수십 개 조합이 가능하므로 엄격 비교보다 학습자 판단이 실용적

## 7. 미결 사항 (CPO 확인 필요)

1. **쓰기 연습에서 뜻(한국어) 타이핑 포함 여부**  
   → 포함 시 IME 정확 비교 복잡. 대안: 뜻은 보여주고 확인만, 영어(단어+동의어)만 타이핑

2. **동의어 정답 기준**  
   → 엄격 일치? 포함 일부? 자가 평가?

3. **동의어 여러 그룹 처리 (`1. ... 2. ...`)**  
   → 그룹별 1개씩? 전체 나열?

## 8. 결론 및 권고안

**권고 설계:**

```
쓰기 연습:
  Step 1: 단어+뜻+동의어 전부 보이는 상태
  Step 2: 영단어 타이핑 → 정확 비교
  Step 3: 동의어 타이핑 → 자가 평가 (정답 표시 후 O/X 선택)

동의어 타이핑 모드 (신규):
  단어 + 뜻 표시 → 동의어 직접 타이핑 → 자가 평가
```

→ 3번 질문의 답변을 받으면 즉시 구현 가능.
